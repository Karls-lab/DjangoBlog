Onto the tutorial! Today, this page is aobut Cascade and Inheritance.

Cascade: When two rules apply that have equal specificity, the one that comes last in the CSS is the one that will be used.

Specificity: The more specific attribute will be applied. A class selector is more specific than an element selector.

Inheritance: Some CSS property values set on parent elements are in herited by their child elements, and some aren't. Properties like widths, margins, padding, and borders do not inherit.

CSS Box Model

Parts of a box

  • Content box: Where the conent is stored, can be sized with width and height
  • Padding box: The white space around the content, can use padding to adjust
  • Border box: The border box wraps the content and any padding.
  • Margin box: The margin box is whitespace applied at the outermost level. Margin is divided up evenly by default, but can be specified with margin-top, right, bottom, or left

Any padding and border is added to the width and height on both sides. Margin is divied up between both sides. Use the DevTools to view the box model. There's such a thing as margin collapsing. If you're not getting the space you were expecting, it might be due to margin collapsing.

Border Border can set border-top, right, bottom, left. There's also border-width, style, color. Theses can also be combined more with border-(direction)-(width, style, or color).

Handling different text directions

writing-mode is a css property that can change the direction of your text. There are three options:

  • horizontal-tb: Top to bottom block flow direction. Sentences run horizontally.
  • vertical-rl: Right to left block flow direction. Sentences run vertically
  • vertical-lr: Left to right block flow direction. Sentences run vertically
